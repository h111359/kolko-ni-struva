## Goal

Fix two categories of data quality defects in the `dim_settlement` dimension table (`data/schema/dim_settlement.csv`) and its Supabase mirror, and ensure every `settlement_name` in the table is unique. Also extend the `dim_settlement` schema with two new columns (`settlement_type` and `oblast`) and reformat all settlement names to a fully-qualified "type name (oblast)" label, applied uniformly to all rows regardless of whether a duplicate existed. Update the React app settlement dropdown sort order to order by base name, then type, then oblast.

1. **Remove 15 duplicate rows (EKATTE deduplication).** Remove 15 rows caused by non-canonical EKATTE formatting. Remap the 30 affected `dim_store.csv` rows from retired settlement keys to the corresponding canonical keys. Remove the retired rows from Supabase `dim_settlement` via a `prune_retired_settlements()` function added to `src/load_supabase.py`.

2. **Add `settlement_type` and `oblast` columns to `dim_settlement`.** For every non-retired row: populate `settlement_type` from the NSI `ek_atte.json` `t_v_m` field; populate `oblast` from `ek_atte.json` `oblast_name` field. For Sofia raion entries (EKATTE code contains "-"): `settlement_type = "čĆ-ąĮ"`, `oblast = ""`. For unknown entries: empty strings.

3. **Reformat all `settlement_name` values to a fully-qualified "type name (oblast)" label** for all ek_atte.json-resolvable rows, e.g., "ą│čĆ. ąÆąĄą╗ąĖą║ąŠ ąóčŖčĆąĮąŠą▓ąŠ (ąŠą▒ą╗. ąÆąĄą╗ąĖą║ąŠ ąóčŖčĆąĮąŠą▓ąŠ)". For Sofia raion entries: use the format "Sofia, čĆ-ąĮ {raion_name}" (e.g., "Sofia, čĆ-ąĮ ąĪčĆąĄą┤ąĄčå" for EKATTE 68134-01, answered by Q005). For unknown entries: keep the current "(unknown:...)" value. After the fix, every `settlement_name` in `dim_settlement.csv` is unique.

4. **Update React app settlement dropdown sort order.** Sort settlement options by: (1) base name (settlement_name stripped of type prefix and oblast qualifier), then (2) settlement_type, then (3) oblast ŌĆö so that "ą│čĆ. ąóčŖčĆą│ąŠą▓ąĖčēąĄ (ąŠą▒ą╗. ąóčŖčĆą│ąŠą▓ąĖčēąĄ)" and "čü. ąóčŖčĆą│ąŠą▓ąĖčēąĄ (ąŠą▒ą╗. ąÆąĖą┤ąĖąĮ)" appear adjacent in the dropdown.

The fix is implemented as a one-time script (`src/repair_dim_settlement.py`) that rewrites both CSVs atomically, followed by updates to `src/load_supabase.py` (schema migration + prune function) and `react-app/src/lib/dataService.js` (settlement fetching + sorting).

After the fix: every EKATTE code in `dim_settlement.csv` satisfies `normalize_settlement_code(ekatte) == ekatte`; no two rows share the same EKATTE; all `dim_store.settlement_key` values reference valid canonical settlement keys; no two rows share the same `settlement_name`; all qualifying rows carry a non-empty `settlement_type` and `oblast`.
