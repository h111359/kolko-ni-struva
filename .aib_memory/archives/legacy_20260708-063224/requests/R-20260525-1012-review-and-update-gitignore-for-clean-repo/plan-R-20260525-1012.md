## Goal

Clean the repository's tracked-file set and close all `.gitignore` gaps so that (a) no secrets or credentials are present in the working tree or future commits, (b) no non-application-code artefacts are tracked, (c) the application is fully restorable from a fresh clone, and (d) git history is rewritten to remove the two committed credential files.

## Constraints

- MUST NOT modify any application source file (`src/`, `react-app/src/`, `tests/`, `menu.py`, `requirements.txt`, `react-app/package.json`, `react-app/package-lock.json`).
- History rewrite uses `git filter-repo`; all three exposed credentials (Netlify token, Supabase service_role key, PostgreSQL password) MUST be rotated by the developer before or immediately after the force-push — this is a manual step that cannot be automated.
- `config.ini` on the developer's machine must be preserved locally; the implementation untracked it and creates `config.ini.example` — it does NOT delete `config.ini` from disk.
- The `.gitignore` negation exception for `data/nomenclatures/` must be placed after the `data/` exclusion line for git to honour it correctly.
- `react-app/node_modules/` removal produces a large commit (~3 300 file deletions from the index); this is expected and unavoidable.
- History rewrite with `git filter-repo` rewrites all commit SHAs; any existing remote branches and open pull requests will be invalidated. The developer must coordinate with all collaborators before force-pushing.

### Task 1: Rewrite git history to remove credential files

#### Intent
Remove `netlify token.txt` and `curl https ekootljybgoenduwprbw.txt` from every historical commit using `git filter-repo`.

#### Outputs
- Git history rewritten (all commit SHAs change).
- Both files absent from all commits.
- `.git/` internal state updated.

#### Procedure
Install `git-filter-repo` if not present:
`pip install git-filter-repo` or `pip3 install git-filter-repo`

Run the history rewrite (from repo root):
`git filter-repo --path "netlify token.txt" --path "curl https ekootljybgoenduwprbw.txt" --invert-paths --force`

Verify both files are absent from HEAD and from an older commit:
`git log --all --full-history -- "netlify token.txt"` — must return no output
`git log --all --full-history -- "curl https ekootljybgoenduwprbw.txt"` — must return no output

Force-push to remote (coordinate with collaborators first):
`git push origin --force --all`
`git push origin --force --tags`

#### Done criteria
- `git log --all --full-history -- "netlify token.txt"` returns no output.
- `git log --all --full-history -- "curl https ekootljybgoenduwprbw.txt"` returns no output.
- Remote reflects the rewritten history (force-push confirmed).

#### Dependencies
- None. Must be performed before Task 3 so the index state after Task 3 reflects the clean working tree.

#### Risk notes
- `git filter-repo` removes the remote `origin` configuration as a safety measure; re-add it after the rewrite: `git remote add origin <URL>`.
- All collaborators must `git clone` fresh or `git fetch && git rebase` — existing local branches cannot be simply `git pull`-ed after a force-push.
- The developer MUST manually revoke/rotate all three credentials immediately after (or before) this step. History rewrite alone does not neutralise credentials if someone already cloned the repo.
