## Goal
Change the landing page data fetching behavior to avoid timeouts by decoupling total row count fetching from data row fetching, resolving the current page immediately.

## Constraints
- Apply changes specifically to the `LandingPage.jsx` logic and its data service interactions.
- Ensure the app stays functional without necessarily showing the total row-count immediately upon filter load.
- Avoid querying the count synchronously with the page rows; the rows must return first.

## Success criteria
- Page rows load without statement timeout errors.
- Total row count is either fetched asynchronously after page data is displayed, or paginated correctly using "next page" logic without needing the total count immediately.
- Test cases are updated to pass with the new deferred-count logic.

## Plan

### Task 1: Update LandingPage data fetching

#### Intent
Modify LandingPage to not wait for the total row count before rendering the page rows.

#### Outputs
react-app/src/components/LandingPage.jsx
react-app/src/services/dataService.js

#### Procedure
Examine `react-app/src/components/LandingPage.jsx` and `react-app/src/services/dataService.js`.

Update `LandingPage.jsx` so that `loadRows` requests the data rows first. Do not use `Promise.all` with `fetchLandingPageCount`.
Set `rows` state as soon as it is fetched. Wait for that to be set, then optionally fetch the count in a separate background effect or don't block the UI rendering on it. For "Do not try to find the total count of the pages immediately and before returning to the user the current page", configure state variables to reflect "calculating..." or defer fetching the count entirely until row data is loaded and displayed.

Provide fallback or dynamic pagination if the count isn't immediately known (e.g. `hasNextPage` logic if rows loaded == page size).

#### Done criteria
LandingPage row data is fetched and rendered without waiting for count.

#### Dependencies
None

#### Risk notes
Total pagination pages might not be known immediately, which we must handle in UI.
