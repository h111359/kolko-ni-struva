## Goal

Eliminate the remaining landing-page first-page `fetchLandingPageRows` statement-timeout failure in the live anonymous Supabase path while preserving the current landing-page browsing behavior. The implementation must also provide validation strong enough to prove the site works in the real runtime path rather than only in mocked local tests.

## Constraints

- The implementation must stay within the existing client-only React-to-Supabase architecture described in `.aib_memory/context.md`.
- The fix must target the current landing-page row path rather than depending on mock-only proof or documentation-only updates.
- Any SQL changes must be provisioned through `src/load_supabase.py` so the repository remains the source of truth for Supabase schema and RPC definitions.
- Validation must include the repository’s automated checks and a real anonymous-path verification after provisioning database changes.
- Documentation updates must cover `.aib_memory/context.md` and any user-facing repository documentation that becomes inaccurate as part of the fix.

## Success criteria

- The landing page loads its first flat-table page without surfacing `fetchLandingPageRows: canceling statement due to statement timeout` for the previously failing case.
- The `get_landing_page_rows` query path in `src/load_supabase.py` is reshaped so pagination happens before wide enrichment work, or an equivalent root-cause fix demonstrably reduces the live first-page cost.
- `react-app/src/lib/dataService.js` and `react-app/src/components/LandingPage.jsx` remain contract-compatible with the refactored row RPC.
- Automated validation covering the touched backend and frontend surfaces passes, including `python -m unittest tests.test_load_supabase` and `cd react-app && npm test && npm run build`.
- A live verification run after `python src/load_supabase.py` confirms that the real landing page loads correctly through the anonymous browser path.
- `.aib_memory/context.md` and any stale repository documentation discovered during the implementation are updated to match the final behavior.
