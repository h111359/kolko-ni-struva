## Goal

Migrate the React frontend Supabase key from the legacy JWT `anon` format (`VITE_SUPABASE_ANON_KEY`) to the new publishable key format (`VITE_SUPABASE_PUBLISHABLE_KEY`) in a hard cutover across all production code, tests, documentation, and configuration templates. Add a frontend prefix validation guard in `react-app/src/lib/supabase.js` that rejects server-side keys (`sb_secret_...`) and JWT-format keys before the Supabase client is instantiated. Document the required Netlify dashboard manual variable rename in `.env.example` and `README.md`. Add a `SUPABASE_SECRET_KEY` placeholder to `.env.example`. Backend ETL is unchanged.

## Constraints

- `src/load_supabase.py` must not be modified; it continues to use `DATABASE_URL` + psycopg2.
- `requirements.txt` must not be changed; `supabase-py` is not introduced.
- `createClient()` in `supabase.js` must retain default call signature with no additional auth options object.
- Hard cutover: no fallback to `VITE_SUPABASE_ANON_KEY` anywhere in the codebase after this change.
- The Netlify dashboard variable rename is a manual operator step; the codebase change only documents it.
- All `VITE_SUPABASE_ANON_KEY` occurrences must be replaced consistently across all 5 affected production files.
- Error messages in `react-app/src/lib/supabase.js` must remain in Bulgarian to match the existing language convention.

## Success criteria

- `grep -r "VITE_SUPABASE_ANON_KEY" . --include="*.js" --include="*.py" --include="*.md" --include="*.example"` (excluding `.aib_memory/` and archived files) returns no matches in production files.
- `react-app/src/lib/supabase.js` exports a non-null `credentialsError` when the key value starts with `sb_secret_`.
- `react-app/src/lib/supabase.js` exports a non-null `credentialsError` when the key value has exactly 3 dot-separated segments (JWT format).
- `react-app/src/lib/supabase.js` exports a non-null `credentialsError` when `VITE_SUPABASE_PUBLISHABLE_KEY` is absent or empty.
- `react-app/src/lib/supabase.js` exports a null `credentialsError` when a non-empty `sb_publishable_...` key is present alongside a valid URL.
- `python -m pytest tests/test_menu.py -v` exits 0.
- `tests/test_menu.py` contains no reference to `VITE_SUPABASE_ANON_KEY`.
- `.env.example` contains `SUPABASE_SECRET_KEY` in the server-side section and `VITE_SUPABASE_PUBLISHABLE_KEY` in the client-side section.
- `README.md` contains a Netlify migration note subsection explaining the required dashboard update with sequencing guidance.
