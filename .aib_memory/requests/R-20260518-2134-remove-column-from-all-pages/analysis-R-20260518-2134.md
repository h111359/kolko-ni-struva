# Analysis: R-20260518-2134

## Overview

**Request ID:** R-20260518-2134
**Request title:** Remove column Ефективна цена from all pages

### Background

The developer has requested removal of the "Ефективна цена" (Effective price) column from all pages of the React analytics app. The effective price is the minimum of a product's retail and promotional price — computed client-side by `calculatePrice()` in `dataService.js` as `min(retail_price, promo_price)` when promo is non-zero, otherwise `retail_price`. The label "Ефективна цена" currently appears in three distinct UI locations: as the eighth of twelve sortable, filterable columns in the compact `FileRowsPanel` table on the Files page; as a field in the `FileRowDetailModal` row-detail modal on the Files page; and as a field in the `RecordDetailModal` record-detail modal on the Report 2 page. No business justification was provided; the request is treated as a UI simplification directive.

### Scope

- Remove the `{ key: 'calculatedPrice', label: 'Ефективна цена', type: 'numeric' }` entry from the `buildColumns()` array in `react-app/src/components/FileRowsPanel.jsx`, reducing the table from 12 to 11 columns.
- Remove the `<dt>Ефективна цена:</dt>` / `<dd>` pair from `react-app/src/components/FileRowDetailModal.jsx`.
- Remove the `<dt style={{ ... }}>Ефективна цена:</dt>` / `<dd>` pair from `react-app/src/components/RecordDetailModal.jsx`.
- Delete the assertion `expect(screen.getByText('Ефективна цена')).toBeInTheDocument();` from `react-app/src/components/FileRowsPanel.test.jsx`.
- Delete the assertion `expect(screen.getByText('Ефективна цена:')).toBeInTheDocument();` from `react-app/src/components/FileRowDetailModal.test.jsx`.
- Update `.aib_memory/context.md` to reflect the reduced column count and the removal of the field from detail modals.

### Out of scope

- The `calculatePrice()` function and `calculatedPrice` data field in `dataService.js` are retained unchanged; they continue to serve Report 2 and Report 3, which display the same value under the label "Цена" (not "Ефективна цена").
- Report 2 table column "Цена" (header `<th>Цена</th>`, value `row.calculatedPrice`) is not changed.
- Report 3 column definition `{ key: 'calculatedPrice', label: 'Цена' }` is not changed.
- The `dist/` build artifacts are not manually edited; they will be regenerated by `npm run build` on next deploy.
- Report 1 is unaffected (it displays an average price bar chart, not a tabular column labeled "Ефективна цена").

### Industry Knowledge

1. **Minimalist design (Nielsen Heuristic #8):** Every element present in a UI competes for the user's attention. Removing a derived metric that is already expressed through adjacent columns (retail price + promo price) reduces cognitive load. The practice of removing redundant derived columns when the source values are already visible is endorsed by Nielsen Norman Group guidelines on data table design and the principle of "minimalist design."

2. **Data–presentation separation in React:** Removing a display column does not require modifying the underlying data contract. In React architecture, data fetching (dataService.js) and presentation (JSX components) are deliberately decoupled. The `calculatedPrice` field continues to exist on every row object returned by the data service; only the view-layer column definition is removed. This separation is consistent with React's component model and the single-responsibility principle.

3. **Test maintenance for display-layer removals:** React Testing Library (RTL) best practice is to test only what is rendered, not what is absent. When removing a UI element, the correct action is to delete the positive presence assertion — not add a negative assertion (`queryByText(...).not.toBeInTheDocument()`). A negative assertion creates a brittle test that could pass for the wrong reason. This is consistent with RTL's "query by role / text" philosophy: only assert the presence of elements that the user can actually see.
