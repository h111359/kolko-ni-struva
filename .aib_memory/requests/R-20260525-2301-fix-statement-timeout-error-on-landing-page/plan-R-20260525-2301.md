## Goal
Rewrite the `get_landing_page_count` RPC function in Postgres to use dynamic SQL in order to execute much faster and avoid statement timeouts natively, keeping exact count functionality.

## Constraints
- Must not change the parameters of the RPC `get_landing_page_count`.
- Dynamic SQL must be used to only conditionally join dimension tables and use filters when they are provided.

## Success criteria
- The `get_landing_page_count` database function is replaced with a dynamically executed version.
- Landing page count is accurate based on the filters.
- Timeout no longer occurs on loading the landing page.
