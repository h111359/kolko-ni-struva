# Input Convention

**Scope:** Normative
**Applies to:** `.aib_memory/input.md` — the active-request input file read and written by all AIB prompts and tool scripts.
**Enforced by:** `.aib_brain/tools/verify-input.py`

---

## 1. Purpose

This document is the authoritative specification for the format rules governing `.aib_memory/input.md`.
`verify-input.py` enforces these rules and exits with code 1 whenever any rule is violated.
Future maintainers MUST update `verify-input.py` whenever these rules change.

---

## 2. YAML Frontmatter Schema

`input.md` MUST begin with a YAML frontmatter block delimited by `---` on the first line and a
closing `---` line. The block MUST contain exactly two top-level nested group keys: `state:` and
`options:`. Each group contains specific sub-keys as defined below.

**`state:` group** — tool-managed fields:

| Sub-key | Type | Allowed values |
| --- | --- | --- |
| `request_id` | string | Any string; `~` when no active request |
| `title` | string | Any string; `~` when no active request |
| `status` | string | `idle` \| `analysis_ready` \| `questions_generated` |
| `input_verification_result` | string or null | `null` \| `valid` \| `invalid` |
| `context_verification_result` | string or null | `null` \| `valid` \| `invalid` |

**`options:` group** — user-configurable fields:

| Sub-key | Type | Allowed values |
| --- | --- | --- |
| `minimum_questions` | integer | Non-negative integer (≥ 0) |
| `input_verification_enabled` | boolean | `true` \| `false` |
| `context_verification_enabled` | boolean | `true` \| `false` |

Rules:
- Only two top-level keys (`state:` and `options:`) are permitted; no additional top-level keys are allowed.
- All sub-keys within each group MUST be present; they MUST be indented with two spaces under their parent group key.
- Boolean values MUST be the lowercase YAML literals `true` or `false` (not `True`, `False`, `yes`, `no`).
- Result flags MUST be the literal `null` (not empty, not `~`, not `None`).
- The workflow state is stored in `state.status` (not as a top-level key named `state`).

---

## 3. Allowed H2 Body Sections

The body of `input.md` (content after the closing `---` delimiter) MUST contain only H2 headings
from the following permitted set. No other `##`-level headings are allowed:

- `## Input` — developer-provided request description; MUST always be present.
- `## Options` — optional developer-supplied processing options (e.g., `Minimum questions:`).
- `## Questions` — Q-blocks generated by `aib-analyze.md`; MUST be present when `state` is `questions_generated`.

---

## 4. Conditional Presence Rule

`## Questions` MUST be present in the body when `state` is `questions_generated`.
When `state` is `idle` or `analysis_ready`, `## Questions` MAY be absent.

---

## 5. Q-Block Format

Each Q-block within `## Questions` MUST conform to the rules defined in
`.aib_brain/conventions/q-block-convention.md`, which is the authoritative source for Q-block
format.

Summary of required Q-block elements (as enforced by `verify-input.py`):

1. **QID line** — first line of the block must match the pattern `**Q###**:` (three-digit
   zero-padded number, e.g. `**Q001**:`, `**Q042**:`).

2. **Why-this-matters line** — must contain a line starting with `> **Why this matters:**`.

3. **Answer options** — the block must contain at least one of:
   - One or more checkbox lines starting with `- [ ]` or `- [x]` (multiple-choice format), OR
   - A `- Answer:` line (free-text format).

4. **Other option (multiple-choice only)** — when checkbox lines are present, at least one line
   containing `Other:` MUST appear in the block (per `q-block-convention.md` requirement that
   all multiple-choice blocks include `- [ ] Other: ___`).

---

## 6. Validation Check List

`verify-input.py` performs the following named checks in order. Each check maps to a rule in this
convention. A check that is skipped due to a prior failure is still counted in the summary.

| Check name | Rule enforced |
| --- | --- |
| `check_frontmatter_present` | File starts with `---` and contains a closing `---` delimiter |
| `check_required_yaml_keys_present` | Two top-level group keys (`state` and `options`) are present; all required sub-keys are present within each group |
| `check_state_value_valid` | `state.status` value is one of `idle`, `analysis_ready`, `questions_generated` |
| `check_minimum_questions_non_negative` | `options.minimum_questions` is a non-negative integer |
| `check_enabled_flags_are_boolean` | `input_verification_enabled` and `context_verification_enabled` (within `options:` block) are `true` or `false` |
| `check_result_flags_valid` | `input_verification_result` and `context_verification_result` (within `state:` block) are `null`, `valid`, or `invalid` |
| `check_allowed_h2_sections_only` | Only `## Input`, `## Options`, `## Questions` H2 headings appear in the body |
| `check_input_section_present` | `## Input` section heading is present in the body |
| `check_questions_section_present_when_required` | `## Questions` is present when `state` is `questions_generated` |
| `check_qblock_format` | Each Q-block in `## Questions` has QID line, Why-this-matters line, answer options, and Other option (if multiple-choice) |

Exit codes: `0` — all checks passed; `1` — one or more checks failed.
